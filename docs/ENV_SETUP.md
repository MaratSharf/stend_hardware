# Настройка переменных окружения

## Безопасное хранение паролей

Начиная с версии 4.6, пароль базы данных вынесен из файла `config.yaml` в переменные окружения.

### Быстрый старт

1. Скопируйте файл `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```

2. Отредактируйте `.env` и укажите реальный пароль:
   ```
   DB_PASSWORD=ваш_пароль
   ```

3. Запустите приложение - пароль автоматически подгрузится из `.env`

### Структура .env файла

```bash
# База данных PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_NAME=stend_db
DB_USER=stend_user
DB_PASSWORD=ваш_пароль_здесь

# Секретный ключ для сессий
SECRET_KEY=уникальная_случайная_строка

# Режим отладки
DEBUG=False
```

### Приоритет значений

Приложение загружает конфигурацию в следующем порядке приоритета:
1. Переменные окружения ОС (наивысший приоритет)
2. Файл `.env`
3. Файл `config.yaml` (наименьший приоритет)

### Безопасность

⚠️ **Важно:**
- Файл `.env` добавлен в `.gitignore` и не должен коммититься в репозиторий
- В production используйте настоящие переменные окружения ОС
- Регулярно меняйте пароль и SECRET_KEY
- Ограничьте доступ к файлу `.env` (chmod 600)

### Migration guide

Если у вас был пароль в `config.yaml`:

**До:**
```yaml
database:
  password: Alexej@12
```

**После:**
```yaml
# config.yaml
database:
  # password: ЗАКОММЕНТИРОВАНО
```

```bash
# .env
DB_PASSWORD=Alexej@12
```
