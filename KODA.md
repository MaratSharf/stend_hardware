# KODA.md — Контекст проекта MV Service v4.5

## Описание проекта

**Стенд машинного зрения (MV Service)** — автоматизированная система контроля качества деталей на основе промышленной камеры Hikrobot и контроллера ОВЕН.

**Версия:** 4.5  
**Последнее обновление:** Апрель 2026

## Запуск

```bash
python run.py
```

**Веб-интерфейс:** http://localhost:5001

## База данных

- **Тип:** PostgreSQL
- **База:** `stend_db`
- **Пользователь:** `stend_user`
- **Файл конфигурации:** `config.yaml`

## Критические исправления

### Flask 2.2+ совместимость

В `web/pages/auth.py` заменено:
- `request.is_xhr` → `request.headers.get('X-Requested-With')`

Изменения в 3 местах:
1. Проверка AJAX-запросов
2. Обработка редиректов
3. Ответы на клиент

## Авторизация и роли

### Роли пользователей

| Роль | Права |
|------|-------|
| `admin` | Полный доступ, включая управление пользователями |
| `naladchik` | Доступ ко всем страницам, кроме управления пользователями |
| `kontroler` | Мониторинг, история, отчёты |
| `operator` | Мониторинг, история |

### Элементы интерфейса авторизации

Все страницы содержат:
- **Сайдбар** с навигацией и блоком текущего пользователя
- **Ссылку "Пользователи"** (👥) в сайдбаре (видна только `admin`)
- **Кнопку выхода** (🚪) в сайдбаре и в правом верхнем углу

### Файлы авторизации

- `web/pages/auth.py` — Flask-маршруты авторизации
- `web/pages/users.py` — управление пользователями
- `web/static/js/auth.js` — клиентская логика (редирект на /auth/login, показ элементов)
- `web/templates/login.html` — страница входа
- `web/templates/_sidebar.html` — общий сайдбар с элементами авторизации

## Основные API

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/status` | Последний результат инспекции |
| GET | `/api/hardware_status` | Состояние входов/выходов |
| POST | `/api/switch_project` | Переключение проекта камеры |
| GET | `/api/tools` | Список инструментов из БД |
| GET | `/api/results` | История результатов |
| GET | `/api/users` | Список пользователей (admin) |
| POST | `/api/users` | Создание пользователя (admin) |

## Примечания для ИИ-агентов

1. **Не предполагайте библиотеки** — проверяйте `requirements.txt` и импорты.
2. **Соблюдайте соглашения** — имена, стили, структура кода.
3. **Тестируйте изменения** — запускайте `python run.py` и проверяйте веб-интерфейс.
4. **Комментируйте редко** — только для сложной логики.
5. **Не пишите болтовню** — краткие, прямые ответы.
