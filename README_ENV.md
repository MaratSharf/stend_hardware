# 🔐 Безопасность: Вынос пароля в .env

## Что изменилось

Пароль базы данных больше **не хранится** в `config.yaml`. Теперь используется безопасный подход с переменными окружения.

## Быстрая настройка

```bash
# 1. Скопируйте пример файла .env
cp .env.example .env

# 2. Отредактируйте .env и укажите ваш пароль
# (файл уже создан с текущим паролем для обратной совместимости)
```

## Структура файлов

```
/workspace/
├── config.yaml          # Основной конфиг (пароль закомментирован)
├── .env                 # Переменные окружения (НЕ коммитить в git!)
├── .env.example         # Шаблон для .env (можно коммитить)
├── .gitignore           # .env добавлен в игнор
└── docs/
    └── ENV_SETUP.md     # Полная документация
```

## Как это работает

1. При запуске приложения `config_manager.py` загружает `.env` файл
2. Переменные окружения (`DB_PASSWORD` и др.) подставляются в конфигурацию
3. Приложение работает как прежде, но пароль защищён

## Преимущества

✅ Пароль не виден в репозитории  
✅ Легко менять для разных окружений (dev/prod)  
✅ Соответствует best practices безопасности  
✅ Обратная совместимость сохранена  

## Для production

В production-окружении задайте переменные напрямую в ОС:

```bash
export DB_PASSWORD="secure-password-here"
export SECRET_KEY="random-secret-key"
```

Или используйте Docker secrets / Kubernetes secrets / Vault.
