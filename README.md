## Назначение

Система предназначена для автоматического визуального контроля деталей на конвейере:

1. **Обнаружение детали** — датчики фиксируют появление объекта.
2. **Позиционирование** — деталь останавливается под камерой.
3. **Съёмка и анализ** — камера делает снимок и определяет качество.
4. **Сортировка** — деталь перемещается к выходу, результат индицируется лампами.
5. **Сохранение результатов** — все проверки записываются в базу данных.

**Результаты:**
- ✅ **OK (годен)** — зелёная лампа, деталь движется к датчику Д4.
- ❌ **NG (брак)** — красная лампа, деталь движется к датчику Д3.

### Поддерживаемые инструменты Hikrobot

Система поддерживает **50+ инструментов машинного зрения** Hikrobot.  
Список инструментов хранится в таблице `tools` базы данных.  
**Особенности:**
- Для инструмента **3.2 CodeRecognition** автоматически отображаются QR-код или OCR-текст.
- Каждый инструмент сопоставлен с проектом камеры (`project_name`), переключение происходит по команде Modbus.
- На странице "Инструменты" вкладка "Все" показывает до 10 последних использованных проектов.

## Оборудование

| Устройство | Модель | Назначение |
|------------|--------|------------|
| Камера машинного зрения | Hikrobot MV-SC3016-06M-WBN | Съёмка и анализ деталей |
| Контроллер ввода-вывода | ОВЕН МК210-312 | Подключение датчиков и исполнительных механизмов |
| Датчики | 6 шт. (Д1-Д4 + 2 тумблера) | Нормально замкнутые (NC) |
| Исполнительные механизмы | 4 выхода | Конвейер, толкатель (резерв), лампы OK/NG |

### Схема подключений ОВЕН

**Входы (DI):**
- DI0 → Датчик Д1 (вход конвейера)
- DI1 → Датчик Д2 (под камерой левый)
- DI2 → Датчик Д3 (под камерой правый)
- DI3 → Датчик Д4 (выход конвейера)
- DI4 → Тумблер (положение A)
- DI5 → Тумблер (положение B)

**Выходы (DO):**
- DO0 → Конвейер (пуск/стоп)
- DO1 → Толкатель брака (резерв)
- DO2 → Лампа NG (красная)
- DO3 → Лампа OK (зелёная)

## Принцип работы

### Конечный автомат сценария C

IDLE → (спад Д1) → CONVEYOR_RUNNING_TO_CAMERA → (спад Д2) → POSITIONED
→ (съёмка) → CONVEYOR_RUNNING_TO_EXIT → (спад Д3/Д4) → IDLE

### Выбор сценария

| Тумблер A | Тумблер B | Сценарий |
|-----------|-----------|----------|
| 1 | 0 | A (непрерывный контроль) |
| 0 | 1 | B (одиночный снимок) |
| 0 | 0 | C (циклический контроль) |

Также возможен принудительный выбор через веб-интерфейс.

## Режимы работы

🟢 **Автоматический режим** (по умолчанию) – работа по заданному алгоритму.

🟡 **Ручной режим** – автоматический алгоритм останавливается, доступно ручное управление через страницу `/debug`.

🔧 **Офлайн-режим** – чекбокс «Офлайн» на главной странице отключает попытки связи с оборудованием (демо-режим). В этом режиме уведомления о потере связи не показываются.

## Установка и запуск

### Требования

- Python 3.8+
- PostgreSQL
- Сетевой доступ к оборудованию

### Установка зависимостей
```bash
pip install -r requirements.txt

Настройка базы данных PostgreSQL

Создайте базу данных и пользователя, укажите параметры в config.yaml:
yaml

database:
  type: postgresql
  host: localhost
  port: 5432
  dbname: stend_db
  user: stend_user
  password: your_password

Веб-интерфейс

Интерфейс выполнен в стиле vdt-service.ru (синяя гамма, шрифт Inter).
URL	Описание

/debug	Ручное управление выходами и триггер камеры
/settings	Редактирование конфигурации, резервные копии
/reports	Формирование отчётов (дневных, сменных и т.д.)
API эндпоинты (основные)
Метод	URL	Описание
GET	/api/status	Последний результат инспекции
GET	/api/hardware_status	Состояние входов/выходов, статус камеры
POST	/api/switch_project	Переключение проекта камеры
GET	/api/tools	Список инструментов из БД
GET	/api/recent_tools	Последние 10 использованных проектов
GET	/api/results	История результатов (с фильтрами)
GET	/api/statistics	Общая статистика

POST	/api/offline_mode	Включение/выключение офлайн-режима
GET	/api/connection_status	Статус связи с оборудованием

Типовые проблемы
Проблема	Возможная причина	Решение
Нет инструментов на /tools	Таблица tools пуста	Выполнить SQL-скрипт для заполнения таблицы
Ошибка подключения к БД	Неверные параметры в config.yaml	Проверить хост, порт, логин, пароль
Изображения не отображаются	Папка data/images недоступна	Проверить права и путь в конфиге
Сообщения о потере связи не появляются	Офлайн-режим включён	Отключить чекбокс «Офлайн»
